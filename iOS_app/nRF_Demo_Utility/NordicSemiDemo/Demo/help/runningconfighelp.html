COMMON SENSOR INFO

Proximity

When set to ON, enables proximity pairing. Default is OFF.

Wildcard

When set to ON, enables iPhone to connect to unpaired devices (devices not in the Manage Connections list). Default is OFF.

Dev ID

Returns the Device ID of the connected sensor.

Signal Eff

Returns the signal efficiency for the connected sensor.

  • For ANT connections, this property represents the ratio of messages received from the sensor over the number of messages expected (range [0, 1.0]). This value should give a good indication of the reception quality for the sensor.
  • For BT Smart connections, this property represents the RSSI (in dBm). If the sensor is not currently connected, the return value is -1.

Sensor Info

Information about the sensor connected.

Manage Connections

List of paired devices. To delete a paired device from the list swipe left on the right hand side of the screen and press Delete.

HEART RATE INFO

beatCount

A single byte value which increments with each heart beat event. This value will roll over every 255 counts.

beatTime

Represents the time of the last valid heart beat event. The time units are 1/1024 seconds. Since the values are stored in two byte integers which have a maximum value of 65536, the value will roll over at 64 seconds (65536 / 1024). This should be taken into consideration when determining the currency of the data.

prevBeat

Represents the time of the previous valid heart beat event.

STRIDE SENSOR INFO

accStride

The accumulated stride count. When available, this value is similar to the strideCount value, but with a larger roll over value.

strideCount

The current stride count. The stride count is a single byte value which rolls over after 255 strides.

unitHealth

The health status of the device. This value uses two bits of data. The values are: 00 OK, 01 Error, and 10 Warning.

latency

The time elapsed between the last speed and distance computation and the current measurement. The resolution of this value is 1/32 second.

useState

The current use state of the device. This value uses two bits of data. The values are: 00 Inactive and 01 Active (the other two values are currently not specified).

moduleLocation

The location of the sensor device. This value uses two bits of data. The values are: 00 Laces, 01 Midsole, 10 Chest, and 11 Other.
